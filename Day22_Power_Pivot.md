Day 22/120 — Power Pivot

Today was all about Power Pivot, and it completely shifted how I see Excel.

Power Pivot is an Excel add-in that manages something called a Data Model — the blueprint that defines how your data is structured and how different tables connect to each other.

What I Explored
1️⃣ Enabling Power Pivot

I enabled the add-in and explored the tabs and features in the Power Pivot ribbon.

2️⃣ Importing Data into the Data Model

I imported data directly into the Data Model, which is different from just storing data in a regular Excel worksheet.
The Data Model allows tables to relate to each other and work together.

3️⃣ Understanding Relationships (This is Where It Clicked 💡)

Power Pivot allows you to connect tables that share related columns.

Think of it like:

Your phone contacts 📱

Your call log 📞

Two separate lists — connected by one thing: a phone number.

Power Pivot does the same with your data tables.

Diagram View

Below is a screenshot of Power Pivot’s Diagram View showing two tables connected through a related column.

4️⃣ Date Tables

I explored how Power Pivot can generate a Date Table with a single click.
It automatically connects to your data, making it easy to filter by:
1. Day
2. Month
3.Quarter
4. Year
This makes time-based analysis much easier and more structured.

5️⃣ Introduction to DAX

I was introduced to DAX (Data Analysis Expressions) — the formula language that Power Pivot unlocks. I didn’t go deep into it today, but I can already see how powerful this is going to be.

I came into today not fully understanding what a Data Model was.

I’m leaving with clarity and a stronger foundation in relational data inside Excel.
